autoConfig

基本信息

Path： /enos-edge/v2.4/discovery/autoConfig

Method： POST

接口描述：

一：点表中baseValue和coefficient说明：
1、如若需要对原始点进行转换，即需要执行ax+b的操作，则需要使用“系数”及“基值”两个参数，其中a是系数（coefficient），b是基值（base-value）；
2、通常情况下不需要修改，直接使用默认值；
3、例如：逆变器电量原始测点数据单位KW，转换为单位W，系数efficiency可以填1000；
二：分连接逻辑：
分连接策略：
1）相同的otDeviceInst由多个enos设备接入，可以在相同的链接下，采集的原始点不允许重复
2）不同的otDeviceInst对接多个enos设备可以在相同的链接下，采集点没有限制，类似otDevice1对1enos设备
3)不同的otDeviceInst被同一个enos设备接入，需要分链接，且不同链接下相同设备的映射模型点不允许重复
入参中deviceConfigs为二维数组，每一个数组元素为同一个连接下的所有设备集合，尽可能的将可以放到同一个连接下的设备放在一起
三：此接口为覆盖式添加接口，每次调用都是全量的接入单个Edge下的所有设备，覆盖删除上次由本接口创建的接入连接和设备，模板(如果没有被其他设备使用的话)

item 类型: object

├─ assetIdstring非必须Edge网关设备assetId
├─ deviceConfigsarray []非必须连接分组信息，每个数组元素代表一个连接下接入的批量设备

├─ 非必须连接下设备与模板的分组关系，每个数组元素代表使用相同模板的一批设备
├─ devicesobject []必须使用相同模板的所有设备列表

item 类型: object

├─ assetIdstring必须网管接入子设备assetId
├─ otDeviceInststring必须otaDeviceInst，例如：10001
├─ modelIdstring必须
├─ pointMappingsobject必须模板的点表映射关系信息

├─ rawPointNamestring必须bacnet搜索测点点名
├─ edgeTypestring非必须

autoConfigV2-modbus

基本信息

Path： /enos-edge/v2.4/discovery/autoConfigV2_1

Method： POST

接口描述：

一：点表中baseValue和coefficient说明：
1、如若需要对原始点进行转换，即需要执行ax+b的操作，则需要使用“系数”及“基值”两个参数，其中a是系数（coefficient），b是基值（base-value）；
2、通常情况下不需要修改，直接使用默认值；
3、例如：逆变器电量原始测点数据单位KW，转换为单位W，系数efficiency可以填1000；
二：数据采集频率配置
uploadType：测点上送频率，
REALTIME:实时上送、
CHANGE：变化上送、
TIMER:定时上送、
NOT_UPLOAD：不上送、
CHANGE_TIMER：变化定时上送
uploadInterval：定时时间间隔，当uploadType为TIMER、CHANGE_TIMER时有效
三：对于规约配置中的registerByteOrder和点表中的valueType配置示例如下
registerByteOrder：寄存器字节序，0大端模式，1小端模式,默认0大端模式
valueType举例INT 和 SW_INT
读取两个寄存器，数据int类型为例：
edge查询：00 00 00 00
00 06 01 03 00 00 00 02
设备应答：00 00 00 00 00 07 01 03 04 11 22 33 44
其中数据段是11 22 33 44

规约配置寄存器字节序registerByteOrder 点表配置数据类型配置valueType 解析后的实际值 说明
大端模式(0) SW_INT 0x11223344 纯大端，高字在前高字节在前
大端模式(0) INT 0x33441122 低字在前高字节在前
小端模式(1) SW_INT 0x22114433 高字在前低字节在前
小端模式(1) INT 0x44332211 纯小端，低字在前低字节在前

四：连接配置全量参数示例
以太网连接：collectType = 0 ModbusTCP规约
"collectConfig":{
"collectName":"AutoCreateModbus",
"connType":"TCP_CLT",
"collectType":0,
"connIP":"1.1.1.11:1101",
"protocolConfigs":{
"registerByteOrder":0,
"callInterval":10000
}
}
串口连接：collectType = 2
"collectConfig":{
"collectName":"AutoCreateModbus",
"collectType":2,
"baudRate":9600,
"checkDigit":0,
"serialNum":"COM1",
"stopDigit":"1",
"protocolConfigs":{
"registerByteOrder":0,
"callInterval":10000,
"crcCheckBitByteOrder":0
}
}

autoConfigV2_bacnet

基本信息

Path： /enos-edge/v2.4/discovery/autoConfigV2_1733799472736

Method： POST

接口描述：

一：点表中baseValue和coefficient说明：
1、如若需要对原始点进行转换，即需要执行ax+b的操作，则需要使用“系数”及“基值”两个参数，其中a是系数（coefficient），b是基值（base-value）；

3、例如：逆变器电量原始测点数据单位KW，转换为单位W，系数efficiency可以填1000；
二：扩展字段说明
interval:若测点需要订阅COV，或者使用sys文件中的查询周期CALL_SUM_INTERVAL，则设置为0,值>0表示该测点采用该值作为查询间隔。只对AI/DI/PI生效，AO/DO测点不读值。
priority:整型，取值范围是1-16和-1，1-16表示正常优先级（默认值16），-1表示忽略优先级。Ø 若没有设置或者超出此范围则取默认值16，仅对AO/DO生效
objectName:如果sys中配置了USE_OBJ_NAME为1，则以逻辑编号中的devName+objName组成的字符串匹配测点与BACnet服务器中的对象
demand:字符串，取值范围“true”或者“false”，不区分大小写, 该值为true时，表示为按需采集点；为其他内容时，表示为普通采集点，按需命令不起作用；按需点用于应用侧需要即时获取数据的场景；应用可通过控制接口，下发按需读取命令
三：数据采集频率配置
uploadType：测点上送频率，
REALTIME:实时上送、
CHANGE：变化上送、
TIMER:定时上送、
NOT_UPLOAD：不上送、
CHANGE_TIMER：变化定时上送
uploadInterval：定时时间间隔，当uploadType为TIMER、CHANGE_TIMER时有效
四：分连接逻辑：
分连接策略：
1）相同的otDeviceInst由多个enos设备接入，可以在相同的链接下，采集的原始点不允许重复
2）不同的otDeviceInst对接多个enos设备可以在相同的链接下，采集点没有限制，类似otDevice1对1enos设备
3)不同的otDeviceInst被同一个enos设备接入，需要分链接，且不同链接下相同设备的映射模型点不允许重复
五：连接信息传参
collectConfig表示Edge接入连接信息，包括连接名称，连接对端地址(bacnet serverIP+port(47808))
相同的serverIp下的所有otDevice的采集映射关系放到configs的一个数组元素内，
分连接逻辑，在原来的基础上，相同的serverIp为一组设备进行连接的拆分，
校验项：
多个serverIP必须相同网段，相同端口，不允许有重复deviceInst
